WHEN TO USE TRY ..EXCEPT


1. Working with External Resources

try:
    file = open("data.txt", "r")
    content = file.read()
except FileNotFoundError:
    print("File not found")
except PermissionError:
    print("Permission denied")

2. Converting Data Types

try:
    user_input = input("Enter a number: ")
    number = int(user_input)
except ValueError:
    print("That's not a valid number!")

3. Making Network Requests

try:
    response = requests.get("https://api.example.com/data")
    data = response.json()
except requests.ConnectionError:
    print("Failed to connect to server")
except requests.Timeout:
    print("Request timed out")

4. Database Operations

try:
    cursor.execute("SELECT * FROM users")
    results = cursor.fetchall()
except mysql.connector.Error as err:
    print(f"Database error: {err}")

5. When You Need to Handle Multiple Error Types

try:
    result = complicated_function()
except ValueError:
    print("Value error occurred")
except TypeError:
    print("Type error occurred")
except Exception as e:
    print(f"Some other error occurred: {e}")

WHEN  NOT use try-except for:

1. Control Flow
Bad:

try:
    if user.is_admin:
        perform_admin_action()
except AttributeError:
    perform_regular_action()
Good:
if hasattr(user, 'is_admin') and user.is_admin:
    perform_admin_action()
else:
    perform_regular_action()

2. Expected Conditions
Bad:

try:
    if len(my_list) > 0:
        item = my_list[0]
except IndexError:
    item = None
Good:
pythonCopyitem = my_list[0] if my_list else None

3 Hiding Programming Errors
Bad:

try:
    result = do_something()
except:  # Never use bare except
    pass
Good:
try:
    result = do_something()
except SpecificException as e:
    logger.error(f"Error occurred: {e}")
    # Handle the error appropriately
Best Practices:

Always catch specific exceptions

# Bad
try:
    do_something()
except:  # Too broad
    pass

# Good
try:
    do_something()
except ValueError:
    handle_value_error()

Use the 'as' keyword to get error details

try:
    do_something()
except ValueError as e:
    print(f"Error details: {e}")

Use else clause for code that should run only if no exception occurs

try:
    data = process_file()
except IOError:
    print("Error processing file")
else:
    print("File processed successfully")

Use finally for cleanup code that must always run

pythonCopyfile = open("data.txt")
try:
    process_file(file)
except Exception as e:
    print(f"Error: {e}")
finally:
    file.close()  # This always runs
